Le site entier, le moteur et chaque résultat de cette section sortent d'un
seul dépôt. Voici comment le mettre en route et refaire les calculs chez vous.

## Récupérer le code

Clonez le dépôt. Tout y est : le moteur Rust, le site web et les sujets de
recherche avec leurs scripts de calcul.

```bash
git clone https://github.com/raphael-anjou/eternity2
cd eternity2
```

## Ce qu'il vous faut

Trois chaînes d'outils. Le moteur est écrit en Rust, le site est une
application Node, et le moteur arrive dans le navigateur compilé en
WebAssembly.

- **Rust**, version stable, avec la cible `wasm32-unknown-unknown`
- **Node.js 22+ et pnpm** pour le site (`corepack enable` installe pnpm)
- **wasm-pack** pour compiler le moteur en WebAssembly
- **just** (facultatif), un petit lanceur de tâches qui enrobe les commandes
  ci-dessous

## Compiler et lancer le site

Avec `just` installé, deux commandes suffisent. La première compile le moteur
en WebAssembly et installe les dépendances du site ; la seconde le démarre sur
`localhost:5173`, avec rechargement à chaud.

```bash
just setup
just dev
```

Vous préférez tout lancer à la main ? Chaque recette `just` n'est qu'un
enrobage d'une ligne, donc rien ne vous oblige à l'utiliser :

```bash
# compiler le moteur en WebAssembly
cd engine && wasm-pack build --target web --out-dir ../web/src/engine/pkg --release
# installer et lancer le site
cd ../web && pnpm install && pnpm dev
```

## Travailler sur le moteur

Le moteur est un crate Rust ordinaire qui se compile aussi en WebAssembly.
Lancez ses tests, puis recompilez le WASM après chaque modification. Les tests
comprennent des vérifications croisées avec de vrais plateaux de la
communauté : ils attrapent le moindre changement du jeu de pièces, des
rotations ou du score.

```bash
just test    # cargo test --release
just wasm    # recompiler le WebAssembly après une modification
```

## Reproduire un résultat de recherche

Chaque sujet de cette section se suffit à lui-même : un article, un script de
calcul et la sortie qu'il produit, archivée dans le dépôt. Relancer le script
régénère cette sortie. Les résultats déterministes reviennent identiques à
l'octet près ; les calculs qui dépendent du hasard ou durent des heures
l'annoncent sans détour, et livrent le plateau trouvé pour que vous puissiez
malgré tout le vérifier dans le visualiseur.

Par exemple, les décomptes des motifs interdits (une vingtaine de secondes) :

```bash
just research-forbidden-patterns
# ou directement :
cd research/topics/forbidden-patterns/compute
cargo run --release > ../results/feasibility.json
```

Lancer `just` sans argument affiche la liste complète des tâches, dont les
autres recettes de reproduction (`research-subgrid`, `research-index`) et les
exécutables natifs du moteur, `bench` et `stats`.

## Lancer toutes les vérifications

Une seule commande rejoue ce que fait l'intégration continue : tests du
moteur, vérification des types, lint et build de production.

```bash
just check   # tests du moteur, typecheck, lint, build
```
